
# Application Screens and Functionalities

## 1. Authentication

*   **Sign In:** Users can sign in using their email and password. Google Sign-In is also supported.
*   **Sign Up:** New users can create an account by providing their email and password.
*   **Forgot Password:** Users can reset their password by providing their email address. A password reset link is sent to their email.
*   **Auth Layout:** A consistent and visually appealing layout is used for all authentication screens.

## 2. Project Management

*   **Project Information:** Users can create new projects by filling out a form with details such as project name, start and end dates, contractor, and consultant.
*   **My Projects:** This screen displays a list of all projects created by the user. Users can view project details and delete projects from this screen.
*   **Home Screen:** The main dashboard of the application, which displays a list of projects. If no projects are available, it provides a prompt to create a new project.
*   **Project Actions:** After selecting a project, users are presented with a menu of actions they can perform, including:
    *   The Basics
    *   Mayor Walls
    *   Tests
    *   Excavation
    *   Ceilings
    *   Reports

## 3. Project Sections

*   **The Basics:** This section is further divided into:
    *   **White Concrete:** A checklist for white concrete work, including pre-handover and works inspection checklists. Users can save the checklist and export it as a PDF.
    *   **Reinforced Foundation Carpentry:** Inspection checklist:
        - Verify that all materials used in the execution of the works are approved.
        - Verify that all drawings required for execution of the works are approved.
        - Verify that the approved method statement for execution of this work item is followed.
        - Verify that all required tests are conducted in accordance with the specifications.
        - Verify that all prerequisite works on which this item depends have been inspected and accepted prior to acceptance.
        - Verify review and compliance with approved design drawings, technical specifications, and other related documents.
        - Verify that all occupational health and safety requirements are met prior to acceptance of the work item.
        - Verify removal of all debris and waste resulting from execution of the works.
    *   **Foundation Reinforcement Steel:** (Content not yet implemented)
    *   **Casting Work:** (Content not yet implemented)
*   **Excavation and Backfilling:** This screen provides checklists for both excavation and backfilling works. Each checklist contains a list of items that need to be verified. Once all items in a checklist are completed, the user can approve the respective work.

## 4. UI Components

*   **Animated Layout:** The application uses animations for screen transitions to provide a smooth user experience.
*   **Reusable Button:** A flexible and reusable button component is used throughout the application with different styles and sizes.
*   **Language Switcher:** The application supports multiple languages (English and Arabic), and users can switch between them using the language switcher.
*   **Theme Toggle:** Users can switch between light and dark themes for better readability and personal preference.
